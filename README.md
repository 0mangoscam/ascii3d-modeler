# WEMWEM v1.2 — Procedural Gaussian Particles

WEMWEM is an experimental Android instrument where sound becomes volumetric procedural particle matter.

## Core rule

No seed. No initial sculpture. No fixed ASCII sculpture style.

Silence is empty. Sound creates 3D Gaussian particle clouds. Those clouds condense into temporary volumetric ASCII matter and dissolve back to black when the sound fades.

## v1.2 changes

- Replaced fixed ASCII-gradient sculpture behavior with procedural glyph matter.
- Added 3D Gaussian particle spawning around moving acoustic attractors.
- Each particle is born with its own random glyph, scale, color, mass, sigma and spin.
- Condensed matter is now rendered as Gaussian ellipsoid clouds instead of pre-shaped tubes/loops.
- Character selection is procedural: dot fields, symbols, blocks, unicode marks and structural glyphs are mixed by energy.
- No permanent initial form. The field remains empty until sound, MIDI or pad input enters.
- MIDI and pad input still generate sound and particle bursts.
- Stability profile from v1.1 is preserved.

## Sound mapping

- Bass creates heavy Gaussian matter.
- Mids create connective clouds.
- Highs create nervous glyph dust.
- Onsets create particle bursts.
- Silence dissolves active particles and condensed clouds.

## Controls

- `LISTEN` enables mic input.
- `VANISH / PERSIST` toggles total disappearance.
- `DECAY` changes disappearance speed.
- `PAD OPEN / PAD CLOSE` shows or hides the touch sound pad.
- Piano row: `A W S E D F T G Y H U J K`.

## Philosophy

WEMWEM should not feel like a visualizer or an ASCII modeler. It should feel like sound opening a probabilistic 3D field where every particle has its own temporary identity.
